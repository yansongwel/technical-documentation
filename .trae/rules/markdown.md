【文档元数据规范】
- 文档顶部必须包含 YAML Front Matter，记录维护信息：
  ```yaml
  ---
  title: [服务名]生产级部署与运维指南
  author: [devinyan]
  updated: [日期]
  version: [文档版本]
  ---
  ```

【文档命名规范】
- 严禁使用 README.md 作为文件名
- 文件名必须使用中文描述性名称，格式：`服务名+生产级部署与运维指南.md`或者`服务名+生产级部署与运维指南-版本号.md`
- 示例：`InfluxDB生产级部署与运维指南.md` 或者 `InfluxDB生产级部署与运维指南-1.28.0.md`

【操作系统兼容性】
- 每个执行步骤须同时提供 Rocky Linux 9 和 Ubuntu 22.04 的命令
- 使用如下格式区分，让用户清晰自选：

  ```bash
  # ── Rocky Linux 9 ──────────────────────────
  dnf install -y <package>

  # ── Ubuntu 22.04 ───────────────────────────
  apt-get install -y <package>
  ```

【高可用架构标准】
- 生产部署章节默认按 3 节点或以上高可用/冗余架构编写
- Master/Primary 节点至少 2 个，保证故障切换

【配置文件创建格式】
- 所有配置文件的创建必须使用如下格式（方便终端直接粘贴执行）：

  ```bash
  cat >> /etc/service/config.conf << 'EOF'
  # 配置内容（含注释）
  key = value   # ← ⚠️ 说明含义，用户根据实际修改
  EOF
  ```

【参数标注规范】
- 用 ★ 标注生产环境"必须修改"的关键参数
- 用 ⚠️ 标注"用户需结合自身环境判断"的参数
- 所有示例 IP、路径、密码均加上 `← 根据实际环境修改` 注释

【文档引用规范】
- 如果部署当前服务时依赖其他服务（如 Nginx、etcd、Harbor 等），
  若已有对应文档，直接用以下格式引用，无需重复编写：
  > 📎 参考：[服务名部署文档](../对应目录/README.md)
