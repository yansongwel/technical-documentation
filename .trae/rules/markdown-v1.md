【全局写作风格】
- **简洁直接**：每句话必须有实质内容，禁止废话和套话（如"部署完成后，我们可以通过以下步骤..."）
- **一事一处**：同一内容只在最合适的章节出现一次，禁止跨章节重复描述
- **章节职责鲜明**：
  - 简介章节：只讲"是什么、能解决什么问题"，不讲安装步骤
  - 部署章节：只写操作命令 + 必要说明，不重复介绍架构
  - 配置章节：只讲参数含义，不重复部署步骤
  - 运维章节：只写运维命令，不重复配置说明
- **特殊情况才说明**：若某步骤在大多数场景下都一样，不加额外解释；只有真正有坑、有差异时才补充说明
- **禁止**：过渡句（"接下来我们..."）、重复摘要（"如上所述..."）、显而易见的注释（`# 安装 nginx`）


【操作逻辑连贯性规范】
- **严格遵循依赖顺序**：任何操作步骤必须先满足前置条件，严禁逻辑倒置。
  - **先创建目录，后写入文件**：在执行 `cat > /etc/xxx/conf` 之前，必须先有 `mkdir -p /etc/xxx` 步骤。
  - **先安装依赖，后配置服务**：严禁配置未安装软件的配置文件（避免路径不存在）。
  - **先创建用户，后分配权限**：严禁对不存在的用户执行 `chown/chmod`。
  - **先配置网络/防火墙，后启动服务**：确保服务启动时环境已就绪。
- **避免"回溯式"指令**：不要在后续步骤中才补充说明"哦对了，之前那个步骤需要先做这个"。所有前置准备必须在主流程中按时间轴顺序提前完成。

【配置文件创建格式】
- 创建新配置文件使用 `cat >`（覆盖写入），追加内容时再用 `cat >>`
- 所有配置文件的创建必须使用如下格式（方便终端直接粘贴执行）：

  ```bash
  cat > /etc/service/config.conf << 'EOF'
  # 配置内容（含注释）
  key = value   # ← ⚠️ 说明含义，用户根据实际修改
  EOF
  ```

【命令幂等性规范】
- 文档中所有 Shell 命令必须保证幂等性（即重复执行不报错、不产生副作用），
  以支持用户在测试环境中多次运行验证脚本。
- 具体要求：
  - 创建目录：使用 `mkdir -p`，而非 `mkdir`
  - 安装软件包：使用系统包管理器（dnf/apt），重复安装会自动跳过，无需额外判断
  - 写入配置文件：使用 `cat >` 覆盖写入，而非 `cat >>` 追加（避免内容重复）
  - 启动服务：使用 `systemctl enable --now`，而非 `systemctl start`（已运行时不报错）
  - 添加用户/角色：命令前加存在性判断，例如：
    ```bash
    id -u appuser &>/dev/null || useradd -r -s /sbin/nologin appuser
    ```
  - 下载文件：下载前检查文件是否已存在，例如：
    ```bash
    [ -f /tmp/package.tar.gz ] || wget -O /tmp/package.tar.gz "https://..."
    ```
  - 执行初始化操作（如数据库 init）：必须加已初始化判断，防止重复初始化损坏数据

【参数标注规范】
- 用 ★ 标注生产环境"必须修改"的关键参数
- 用 ⚠️ 标注"用户需结合自身环境判断"的参数
- 所有示例 IP、路径、密码均加上 `← 根据实际环境修改` 注释

【环境差异标注规范】
- 凡是配置文件中存在因部署环境不同（物理机 / 虚拟机 / 云主机）而导致参数值不同的项，
  必须用行内注释列出各场景的推荐值或注意事项，让读者自行判断选择。
- 使用 🖥️ 符号标注"依赖环境类型"的参数，格式如下：

  ```yaml
  # 🖥️ 【环境差异】根据实际部署环境选择：
  #   - 物理机：直接使用物理网卡名（如 eth0/ens3），性能最优
  #   - 虚拟机（VirtualBox/VMware）：使用 eth1 或 ens33，注意网卡绑定
  #   - 云主机（阿里云/AWS/腾讯云）：使用内网 IP，禁用 IPIP 协议（云平台拦截）
  network_interface: "eth0"   # ★ ← 根据 ip addr 命令查到的实际网卡名填写
  ```

- 常见需要环境差异标注的参数场景（必须覆盖）：
  - **网卡名称 / 绑定 IP**：物理机固定名称、虚拟机可能是 eth1/ens33、云主机需查控制台
  - **存储路径 / 磁盘挂载**：物理机多块裸盘、虚拟机附加磁盘、云主机挂载数据盘路径不同
  - **网络模式（CNI/Calico 等）**：云主机推荐 VXLAN，物理机推荐 BGP，虚拟机均可
  - **内存 / CPU 限制参数**：根据实际节点规格调整，不应写死固定值
  - **时钟同步源（NTP）**：内网环境用内部 NTP 服务器，外网用公共 NTP
  - **防火墙策略**：云主机通常在安全组控制，不需要 firewalld；物理机需要
